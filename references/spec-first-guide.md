# Spec-First 完整指南

## 定位

Spec-First 不是一种工具，而是一种理念/哲学。它是 Spec-Kit、OpenSpec、Superpowers 的共同底层思想。

**核心**：在写第一行代码之前，先把以下四件事写清楚：
1. **做什么（What）**
2. **为什么做（Why）**
3. **约束（Constraints）**
4. **非目标（Non-Goals）**

## 适用场景

- **所有人**——不需要安装任何工具
- 想先感受 SDD 是什么，不想装工具
- 环境限制无法安装 Spec-Kit/OpenSpec/Superpowers
- 快速验证 spec-first 理念是否适合自己

## 最小可行方案：AGENTS.md

不想装任何额外工具？用 spec-first 理念完全可行：

### 1. 创建 AGENTS.md

在项目根目录维护一个 `AGENTS.md` 文件：

```markdown
# AGENTS.md

> 本项目规范文件。AI 开发前必须先阅读此文件。

## 项目概述
- **项目名**：[名称]
- **技术栈**：[前端/后端/数据库]

## 技术约束
- [约束 1]
- [约束 2]

## 质量要求
- [要求 1]
- [要求 2]

## 非目标（v1 不做）
- [ ] [非目标 1]
- [ ] [非目标 2]
- [ ] [非目标 3]

## AI 行为规范
1. 先读此文件，再开始工作
2. 严格遵守 Non-Goals
3. 所有代码必须通过测试
```

### 2. 需求模板

每次开新会话，先粘贴需求模板，把四要素填完：

```markdown
## 功能：用户登录

### What
用户可以用邮箱+密码登录，登录成功返回 JWT token

### Why
现在没有认证机制，所有接口都是裸露的

### Constraints
- 密码必须 bcrypt 加密
- 连续 5 次失败锁定 15 分钟
- token 有效期 7 天

### Non-Goals（v1 不做）
- OAuth 第三方登录
- 手机号登录
- "记住我"功能
```

### 3. 会话分离

**会话 A（规划）**：
- 让 AI 读取 AGENTS.md
- 粘贴需求模板，填写四要素
- 让 AI 生成详细 spec（保存为 `specs/功能名.md`）
- 确认 spec 后结束会话

**会话 B（执行）**：
- 让 AI 读取 AGENTS.md + specs/功能名.md
- 按 spec 写代码
- 定期回查 spec

## 优点

- **成本最低**：不需要安装任何工具
- **效果立竿见影**：即使只做 AGENTS.md + 需求模板，也能防止 60% 的返工
- **工具无关**：任何 AI 工具、任何编辑器都能用
- **渐进式**：可以先从 AGENTS.md 开始，觉得好用再升级工具

## 缺点

- 不强制执行（靠自觉）
- 没有自动化工具辅助
- 没有版本归档机制（需要手动维护）

## 升级路径

```
阶段 1：AGENTS.md + 需求模板（今天就能开始）
    ↓
阶段 2：加入 OpenSpec（增量规范管理）
    ↓
阶段 3：加入 Superpowers（TDD + 代码审查）
    ↓
阶段 4：全面 Spec-Kit（宪法式规范管理）
```

**建议**：不要一开始就上 Spec-Kit，先从 AGENTS.md 开始，跑通 2-3 个功能后，再根据需要升级工具。
