## 5. 接口设计（API Design）

### 5.1 接口列表

```
POST /api/v1/auth/login
```

**请求体：**
```json
{
  "email": "string, required, 邮箱格式",
  "password": "string, required, 8-64 字符"
}
```

**成功响应（200）：**
```json
{
  "success": true,
  "data": {
    "token": "string, JWT token",
    "expires_in": 604800
  }
}
```

**错误响应：**
```json
// 400 - 参数错误
{
  "success": false,
  "error": {
    "code": "INVALID_INPUT",
    "message": "请求参数错误",
    "details": ["邮箱格式不正确"]
  }
}

// 401 - 认证失败
{
  "success": false,
  "error": {
    "code": "AUTH_FAILED",
    "message": "邮箱或密码错误"
  }
}

// 429 - 登录频繁
{
  "success": false,
  "error": {
    "code": "TOO_MANY_ATTEMPTS",
    "message": "登录尝试过于频繁，请 15 分钟后重试"
  }
}
```

### 5.2 状态码约定
- 200：成功
- 400：请求参数错误
- 401：认证失败
- 429：请求过于频繁
- 500：服务器内部错误
