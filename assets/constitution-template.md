# 项目宪法 / Project Constitution

> **不可违背原则**：本文件是项目的最高约束。所有 Spec（功能规格、变更规格、执行计划）和所有代码都必须遵守本文件的每一条规则。
> 修改本文件需要团队集体决策，AI 不得擅自修改。

> 生成日期：YYYY-MM-DD
> 版本：v1.0

---

## 1. API 设计

### 1.1 规范
- [ ] 所有 API 遵循 RESTful 规范，版本化路径（`/api/v1/...`）
- [ ] 错误响应统一使用 [RFC 7807 Problem Details / 自定义格式]
- [ ] 所有 API 必须有 OpenAPI 3.0 / 等效文档

### 1.2 版本策略
- [ ] 向后兼容的变更：小版本递增
- [ ] 破坏性变更：大版本递增，旧版本至少维护 [N] 个月

---

## 2. 安全

- [ ] 所有用户输入必须经过校验和清洗
- [ ] 敏感数据（密码、Token、密钥）禁止出现在日志中
- [ ] 数据库查询必须使用参数化查询，禁止字符串拼接
- [ ] 认证方式：[JWT / Session / OAuth2 / ...]
- [ ] 密码加密：[bcrypt / argon2 / scrypt]，cost factor = [N]

---

## 3. 代码质量

- [ ] 单元测试覆盖率不低于 [80]%
- [ ] 所有公共方法必须有文档注释 / TypeScript 类型定义
- [ ] 禁止引入未经安全审计的第三方依赖
- [ ] 代码风格：[ESLint / Prettier / Ruff / ...]，配置与 CI 绑定

---

## 4. 基础设施

- [ ] 所有服务必须支持优雅关闭（Graceful Shutdown）
- [ ] 配置项通过环境变量注入，禁止硬编码
- [ ] 日志格式统一：[JSON / 结构化文本 / ...]
- [ ] 监控与告警：[Prometheus + Grafana / Sentry / ...]

---

## 5. 本宪法的使用方式

1. **项目初始化时创建**：团队在项目启动时共同确定本条宪法
2. **所有 Spec 默认继承**：每个 Spec 无需重复声明宪法已包含的约束
3. **Spec 与宪法冲突时，宪法优先**
4. **AI 行为规范**：AI 在开始任何开发工作前必须先加载本文件；遇到不确定的设计决策时，以本文件为最高依据
5. **定期回顾**：[每月 / 每季度] 回顾一次，更新过时约束，版本递增
